# Pertemuan 4 — UI/UX Design, Wireframing, dan Prototyping

**CPMK51 · Design thinking · Usability heuristics · Wireframing · Prototyping**

## Capaian Pembelajaran

Setelah mengikuti pertemuan ini, mahasiswa mampu:

1. Menjelaskan perbedaan UI dan UX dalam aplikasi web.
2. Menerapkan design thinking secara ringkas untuk studi kasus Asisten AI untuk UMKM.
3. Membuat persona sederhana dan user flow utama.
4. Membuat wireframe low-fidelity untuk minimal tiga halaman.
5. Membuat prototype klik menggunakan Figma atau alat sejenis.
6. Mengevaluasi rancangan menggunakan checklist usability dan aksesibilitas.

## Materi

- UI vs UX.
- Design thinking: empathize, define, ideate, prototype, test.
- Persona dan kebutuhan pengguna.
- User flow.
- Wireframe low-fidelity.
- Prototyping dengan Figma.
- Usability heuristics dan aksesibilitas dasar.
- Integrasi AIK: Al-‘Adl, Rahmah, dan Amanah dalam desain antarmuka.

## Aktivitas Praktikum

Mahasiswa membuat rancangan awal untuk aplikasi **Asisten AI untuk UMKM**:

1. Menentukan persona pengguna.
2. Menulis tujuan pengguna.
3. Membuat user flow.
4. Membuat wireframe 3 halaman: Beranda, Form Produk, Hasil Caption AI.
5. Membuat prototype klik.
6. Menguji rancangan dengan checklist.
7. Menulis refleksi desain.
